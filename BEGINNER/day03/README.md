# Day 03 — Conditionals: if/else, switch, ternary

## Objectives
- Control program flow based on conditions
- Distinguish if/else chains from switch statements
- Use the ternary operator for concise expressions
- Understand switch fall-through and when to use it

## Concepts Covered
- `if / else if / else`
- Nested conditionals
- `condition ? true_val : false_val` (ternary)
- `switch / case / default / break`
- Intentional fall-through in switch
- Common mistake: `=` vs `==` in conditions
